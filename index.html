📖 新手教學 / 懶人包

這個 App 是「出貨雙重確認」:刷寄件單 → 刷/點商品計數 → 出貨完成,避免漏裝、錯裝。點下面標題展開看。

📦 一般出貨(蝦皮 CSV)
1. 上方選通路(蝦皮)→ 上傳當天揀貨單 CSV。
2. 第一次出現的規格要點一次選對應 SKU,之後自動記住。
3. 開「📦 裝箱掃描」→ 刷寄件單號開始一張單。
4. 刷商品 → 數字 +1;一張單裝完刷下一張單號即可。
5. 全部包完 → 按「完成出貨」→ 存入紀錄、自動扣庫存。
💡 倍數:輸入 *5 再刷,一次 +5。
👥 共同包貨(兩人一起)
1. 兩台都確認 ⚙ 設定最下面是最新版本號、暱稱有設(別取一樣)。
2. 主機台:匯入 CSV 建好清單 → 開房 → 拿到 4 碼。
3. 另一台:先把自己沒完成的單收掉 → 輸入 4 碼加入。
4. 兩台徽章顯示「2👤」→ 各刷一筆測試、確認對方看得到 → 取消測試單。
5. 各包各的(大數字只顯示自己包的,正常)。
6. 收工:只由一人按「完成出貨」,按前確認兩台都沒在 ⚠。
⚠ 徽章 ⚠ = 重連中,稍等變回房號再刷(刷的會先存著)。
☁ 看到彼此的出貨紀錄(Spidey Key)
兩件事要分清楚:
• 共同包貨(一起刷商品)=房間,不用金鑰,填 Worker URL + 暱稱、輸 4 碼就能一起刷。
• 共享出貨紀錄 + 盤點紀錄(看彼此完成的單 / 手機盤完電腦看得到)=雲端,一定要 Spidey Key。
要看到同仁的出貨紀錄:兩台都要在 ⚙ 外部整合 →「X-Spidey-Key」欄位填同一組(跟 Worker 設定的一樣),完成的單才會上雲共享。
• 只有一台有填 → 你看得到自己的、看不到對方的(對方沒上雲)。
• 填好後回出貨紀錄按「重新整理」拉一次。
測試跳「連線失敗 HTTP 401」 = 金鑰沒填或填錯(不是 URL 錯);把兩台 Spidey Key 對成一模一樣即可。
💡 只想一起刷商品、不需要共享紀錄的話,401 可以不理,刷商品照常運作。
📮 補寄 / ↩️ 退貨 / 🏭 入倉
• 補寄:售後補件。客人手機選填。不扣庫存、不推。
• 退貨確認:收回商品,依原因分倉——良品→一般倉、商品瑕疵→報廢倉、商品故障→檢修倉。
• 入倉驗收:進貨。填進貨單號 → 刷/點商品。完成後加進一般倉庫存,不推。
💡 連續多位客人:刷下一位單號會跳「換下一位?」自動收上一筆。
⚠ 缺貨標記
某商品沒庫存、這批出不了時,在備貨清單該商品下面點「標記缺貨」→ 那列變紅、出現 ⚠ 缺貨。
• 你照常刷貨,要不要標缺貨自己決定(不會自動扣庫存)。
• 標了之後,當前訂單那項也會變紅顯示,包的時候不會誤記。
• 出貨完成的報告和出貨紀錄都會列出缺貨清單,事後查得到。
• 共同包貨:兩台會即時同步缺貨標記。建議由開房者(收尾那台)標,紀錄以收尾那台為準。
🖨 刷槍條碼列印
商品庫 → 列印 SKU 條碼。預設都不勾(避免浪費紙);要印哪幾個就勾哪幾個,或按「✅ 全選」。集中貼在包貨桌,刷槍掃 = 當前訂單該商品 +1。
💾 資料安全(很重要)
• 資料存在這台手機/電腦本機。換機、清快取會不見。
• 請到 ⚙ → 外部整合 設定 Discord 自動備份:每次出貨完成會自動存一份。
• 換手機 / 換網址:用 ⚙ → 資料搬家/備份 匯出 JSON,在新裝置匯入。
⚠ 沒設備份時,開 App 會跳提醒。
❓ 常見問題 / 紅旗
• 某商品清單裡沒有 → 用最新版重新匯入當天 CSV。
• 看不到同仁的出貨紀錄 → 兩台的 Spidey Key 要填同一組(見上面「☁ 看到彼此的出貨紀錄」)。
• 連線失敗 HTTP 401 → 是金鑰沒填/填錯,不是 URL 錯;共同包貨刷商品不受影響。
• 數字對不上 → 看徽章是否 ⚠;兩台版本是否一致。
• 白畫面 / 打不開 → 先用備份還原,並回報。
• 連動商品 / 倍數失效 → 回報。
找不到答案就截圖回報。

3. 貼以下程式碼: 

function doPost(e) {
  const d = JSON.parse(e.postData.contents);
  const sheet = SpreadsheetApp
    .getActiveSpreadsheet()
    .getActiveSheet();
  if (d.type === "picking_outbound") {
    sheet.appendRow([
      new Date(),
      d.doc_no, d.customer,
      d.items.map(i =>
        i.sku + "x" + i.qty
      ).join(", ")
    ]);
  }
  return ContentService
    .createTextOutput("OK");
}

💾 資料搬家 / 備份

把整個 App 的資料 (商品庫 + 庫存 + 規格對應 + 出貨紀錄 + 設定) 存成一個 JSON 檔, 之後可以在另一個網址 / 手機匯入。

⚠ 瀏覽器資料依「網址」分開。例如 koca-picking.pages.dev 跟 xxx.workers.dev 是兩個獨立的資料盒, 互相看不到。換網址前先匯出, 在新網址匯入, 才能延續。

包含哪些資料?
  • 商品庫 (SKU + 名稱 + 分類 + 條碼 + 必要商品標記)
  • 倉庫清冊 (75 SKU 的現有庫存)
  • 規格 → SKU 對應 (蝦皮揀貨單用)
  • 出貨紀錄 (一般 + 補寄)
  • 設定 (聲音 / Webhook / 主題)
  • 當前未完成的備貨清單 (state)
